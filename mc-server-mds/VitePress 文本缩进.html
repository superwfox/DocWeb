ChatGPT 说：

在 VitePress 里“让所有普通文本默认前面空两格”，需要先明确你说的“空两格”是哪一种语义，否则实现方式会完全不同：

  • A. 视觉上的首行缩进 2 个汉字（常见写作排版）：用 CSS 实现，推荐。

  • B. 源码里每一段都自动插入两个空格字符：这不建议（会破坏 Markdown 语义、列表/引用/代码块等），也很难做到“只影响普通文本”。

下面按最常见的 A（首行缩进） 给出可直接落地的方案，并尽量只作用于“普通段落”。

方案 1：全站普通段落首行缩进（推荐）

在你的 VitePress 项目中新建或编辑：

docs/.vitepress/theme/custom.css

加入：

css
/* 只处理正文区域的普通段落 */
.VPDoc .content p {
  text-indent: 2em; /* 2em 约等于 2 个汉字宽度 */
}

然后在 docs/.vitepress/theme/index.ts（或 .js）引入：

ts
import DefaultTheme from 'vitepress/theme'
import './custom.css'

export default DefaultTheme

效果：所有正文区域的 <p> 段落首行缩进 2em。

方案 2：避免影响“提示框/表格/列表”等非正文段落（更精细）

有些组件（比如提示框、卡片、评论区等）内部也会有 <p>，你可能不想缩进它们。可以更精细一些，例如仅对“Markdown 正文产生的段落”生效（仍然基于 DOM 结构经验做选择器约束）：

css
/* 更保守：只在 doc 内容主容器中缩进段落 */
.VPDoc .vp-doc > p {
  text-indent: 2em;
}

如果你的主题结构是默认主题，一般 vp-doc 是 Markdown 正文容器；这条规则通常比 .content p 更不容易误伤组件内部的段落。

重要注意事项（你大概率会遇到）

  1. 第一段是否也缩进？
    如果你想“第一段不缩进、后续段落缩进”，可以用相邻兄弟选择器：

    css
    .VPDoc .vp-doc > p + p {
  text-indent: 2em;
}

  2. 标题后的第一段要不要缩进？
    通常标题后的第一段不缩进更舒服，可以这样：

    css
    .VPDoc .vp-doc > h1 + p,
.VPDoc .vp-doc > h2 + p,
.VPDoc .vp-doc > h3 + p,
.VPDoc .vp-doc > h4 + p,
.VPDoc .vp-doc > h5 + p,

如果你的目标文件夹路由前缀固定，比如 docs/guide/ 对应访问路径是 /guide/xxx，可以用“当前页面链接高亮项”来间接限定（不同版本/主题 DOM 可能略有差异，所以不如方案 1 稳）：

css

.VPSidebarItem.is-active a[href^="/guide/"] ~ * /* 这类写法往往要结合实际 DOM 调整 */ {
  /* 很多时候无法直接选到正文容器，所以此路不一定通用 */
}

因为 CSS 很难直接获取“当前路由”，除非你有额外 class 或属性可依赖，所以这类纯 CSS 路由判断通常不可靠。

方案 3：在该文件夹使用不同布局/主题注入 class（适合你想自动化）

如果你不想手工给每篇 md 加 frontmatter，可以做一个小自动化：自定义主题 Layout，根据 page.relativePath 判断是否在某文件夹下，动态给 body 加 class。

docs/.vitepress/theme/Layout.vue（示意）：

vue
<script setup>
import DefaultTheme from 'vitepress/theme'
import { useData } from 'vitepress'
import { onMounted, onBeforeUnmount, watchEffect } from 'vue'

const { page } = useData()

const cls = 'indent2'
const targetPrefix = 'your-folder/' // 例如 'guide/'，相对 docs 根目录

const apply = () => {
  const inFolder = page.value.relativePath.startsWith(targetPrefix)
  document.body.classList.toggle(cls, inFolder)
}

onMounted(() => {
  watchEffect(apply)
})
onBeforeUnmount(() => {
  document.body.classList.remove(cls)
})
</script>

<template>
  <DefaultTheme.Layout />
</template>

然后 CSS 仍然用：

css
body.indent2 .VPDoc .vp-doc > p {
  text-indent: 2em;
}

优点：一次配置，该文件夹下所有 md 自动生效；缺点：需要维护一个自定义 Layout（但代码很小）。

watchEffect(() => {

严格来说，“教堂”没有一个在现代汉语里通用、无歧义的“一字简称”。一字写法往往会引起歧义（因为同一个字在别的场景也常用）。

不过在特定语境里，确实有人会用一字来“省略”：

  • 堂：最常见的省略写法，来自“圣堂/礼拜堂”。

    • 优点：在宗教建筑语境里比较自然。

    • 缺点：歧义很大（学堂、食堂、礼堂、药堂等），脱离上下文容易误解。

  • 会：更偏向“教会/堂会”的“会”。

    • 优点：强调组织而非建筑。

    • 缺点：更容易和“协会/会议”混淆，也不等同于“教堂”这栋建筑。

如果你是写作或做标题、地图点位那种“极短标签”，通常更稳妥的是用 “堂”，但最好第一次出现时用全称（“教堂”或“圣堂”），后文再用“堂”省略。
